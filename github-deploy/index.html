Security Architecture

Built for real privacy

Every security property is enforced cryptographically — not by policy, not by trust.

🔑

Browser-side Key Generation

ECDH P-256 key pairs are generated locally using the Web Crypto API. Private keys never leave your device.

🛡️

AES-256-GCM Encryption

Each message is encrypted with a unique IV before it leaves your browser. Provides both confidentiality and integrity.

🔒

Zero-Knowledge Server

The server stores only ciphertext blobs. Without your private key, messages are mathematically unreadable.

🛡

Replay Attack Protection

Every message carries a UUID v4 and timestamp. The server enforces uniqueness and rejects messages outside ±5 minutes.

🔐

PBKDF2 Key Wrapping

Your encrypted private key is stored server-side, wrapped with a key derived from your password (100,000 iterations).

⚡

Real-time via Socket.IO

Messages are relayed instantly over WebSockets. The server only sees encrypted payloads — never plaintext.

🔏

TOFU Fingerprint Verification

SHA-256 public key fingerprints are displayed per contact. Alerts you the moment a key changes (possible MitM).

🚫

Session Fixation Prevention

Session IDs are regenerated after every login. Cookies are HttpOnly, Secure, and SameSite=Strict.
